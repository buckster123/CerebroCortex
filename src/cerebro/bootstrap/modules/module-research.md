# Module: research
## Purpose
Research and synthesis patterns for literature review, domain exploration, and knowledge building.

## Activation Signals
- "paper", "arxiv", "study", "research", "state of", "survey", "review", "literature"
- "what is the latest", "who wrote", "how does X work", "explain the theory"
- "compare approaches", "benchmark results", "evaluation", "dataset"

## Cognitive Patterns
1. **Source triangulation** — Never rely on a single source. Cross-reference at least 3.
2. **Primary sources first** — Prefer papers over blog posts, official docs over tutorials.
3. **Synthesis over summary** — Don't just list findings — connect them, find patterns, identify gaps.
4. **Recency bias check** — Newer is not always better. Check citation count and methodology.
5. **Reproducibility lens** — Can this result be replicated? Is code/data available?
6. **Store findings** — Use `remember()` with ["research", "paper", "domain:X"] tags for future recall.

## Inhibitions
- Never present a paper's abstract as the full story — read the methodology.
- Never ignore contradictory findings — science thrives on conflict.
- Never skip linking related research memories — build a knowledge graph.

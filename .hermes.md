# CerebroCortex — Project Context

Brain-analogous AI memory system. PyPI: `cerebro-cortex`. MCP server + REST API + CLI + dashboard.

## Architecture
- **6 memory types:** episodic, semantic, procedural, affective, prospective, schematic
- **9 brain-region engines:** Thalamus (gating), Amygdala (affect), Temporal Lobe (semantics), Hippocampus (episodes), Association Cortex (linking), Cerebellum (procedural), Prefrontal Cortex (executive), Neocortex (schemas), Default Mode Network (dream)
- **Triple storage:** SQLite (canonical) + igraph (in-memory traversal) + ChromaDB (vectors). VisionVectorStore for CLIP embeddings.
- **Activation:** ACT-R power-law decay + FSRS spaced repetition + spreading activation (Collins & Loftus)
- **Scoring:** 35% vector + 30% activation + 20% retrievability + 15% salience
- **Multi-process sync:** igraph auto-resyncs from SQLite via `PRAGMA data_version`. Zero polling, zero locks.

## Development Rules
- **Always use venv:** `venv/bin/python` — system python lacks deps.
- **Tests:** `venv/bin/python -m pytest tests/ -v` — expect 538+ passing.
- **Schema changes:** Bump `SCHEMA_VERSION`, add idempotent migration block.
- **New MCP tools:** Add to `TOOL_SCHEMAS` + `call_tool()` dispatch in `mcp_server.py`. Update count in `test_mcp_server.py`.
- **New cortex methods:** Follow `remember()`/`recall()` pattern — enrichment pipeline for writes, ranking pipeline for reads.
- **ChromaDB metadata:** Keep flat (no nested dicts). Add fields to `node_to_metadata()` in `chroma_store.py`.
- **Graph pruning:** Only nodes with `degree == 0` are pruned — bridge nodes are safe.
- **Gating bypass:** `send_message()` and `store_intention()` skip Thalamus.

## Multi-Agent Setup
3 registered agents share one store: CLAUDE-OPUS (this host), CLAUDE-HAILO (Hailo Pi), CLAUDE-APEX (ApexAurum cloud). Cross-instance via `send_message`/`check_inbox` — stored as SEMANTIC with `from:{agent}`/`to:{agent}` tags.
